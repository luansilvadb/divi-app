# UI/UX Refactor: PrimeVue-First Architecture

**Goal:** Refatorar a UI/UX do Divi Finance para utilizar EXCLUSIVAMENTE PrimeVue como library principal, eliminando wrappers desnecessários, maximizando performance, coesão e densidade lógica, assegurando baixo acoplamento e preservação de contratos existentes (retrocompatibilidade).

**Current State Analysis:**

- Vue 3 + TypeScript + Vite
- PrimeVue 4.5.4 já instalado e configurado
- Tailwind CSS 4.2.2 como utility-first CSS
- Custom Atomic Design wrappers em `src/shared/components/`: atoms (8), molecules (5), organisms (9), templates (2)
- Custom theme preset (`diviPreset.ts`) extendendo Aura
- Problema: Duplicação de responsabilidades - wrappers customizados reimplementam funcionalidades já presentes no PrimeVue

**Target Architecture:**

- PrimeVue como única fonte de componentes UI
- Tailwind CSS apenas para layout utilities e customizações não-cobertas pelo PrimeVue
- Eliminação progressiva de wrappers redundantes
- Preservação de interfaces/contracts existentes para retrocompatibilidade
- Aproveitamento máximo do theming system do PrimeVue 4

## Phase 3: Component Migration Strategy

### Task 3.1: Eliminate Trivial Wrappers (High Value/Low Risk)

**Files:**

- `src/shared/components/atoms/BaseButton.vue` → Usar `primevue/button` diretamente
- `src/shared/components/atoms/BaseSkeleton.vue` → Usar `primevue/skeleton` diretamente
- `src/shared/components/atoms/BaseBadge.vue` → Usar `primevue/badge` diretamente
- `src/shared/components/atoms/BaseProgressBar.vue` → Usar `primevue/progressbar` diretamente

**Migration Pattern:**

1. Criar script de migração de imports
2. Substituir `<BaseButton>` → `<Button>` em todos os arquivos
3. Mapear props custom para props PrimeVue equivalentes
4. Atualizar preset para incluir estilos atualmente nos class do wrapper

**Retrocompatibilidade:** Exportar aliases em `src/shared/components/index.ts` se necessário para transição gradual

### Task 3.2: Refactor Smart Wrappers (Medium Value/Medium Risk)

**Files:**

- `src/shared/components/atoms/BaseCard.vue` → Manter wrapper mas simplificar usando slots PrimeVue nativos
- `src/shared/components/atoms/BaseInput.vue` → Unificar com PrimeVue InputText/InputNumber/IconField
- `src/shared/components/atoms/BaseSelect.vue` → Validar se Select do PrimeVue cobre todos os casos

**Action:** Reduzir complexidade mantendo funcionalidades específicas (error states, loading, empty)

## Risk Mitigation

| Risk                    | Mitigation                                     |
| ----------------------- | ---------------------------------------------- |
| Breaking changes        | Manter wrappers como aliases durante transição |
